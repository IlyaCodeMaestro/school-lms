# school-lms

Портфолио-проект, который демонстрирует ленту активности школы с авторизацией, управлением постами и аналитикой прогресса. Интерфейс локализован на русском языке и оптимизирован под десктопы и мобильные устройства, также присутствует pixel perfect верстка.

## Основные функции
- Авторизация и регистрация через `zustand`-хранилище с мок-пользователями.
- Лента достижений (`components/post-card.tsx`) с фильтрами по ученикам, учителям и предметам (`components/filter-sidebar.tsx`).
- Создание и редактирование публикаций через модальные окна (`components/create-post-modal.tsx`, `components/edit-post-modal.tsx`) с валидацией форм.
- Адаптивная навигация: вкладки, мобильное меню, профиль пользователя и тост-уведомления (`hooks/use-mobile.ts`, `hooks/use-toast.ts`).
- Набор тестов на Vitest + Testing Library (`tests/*.test.tsx`) для критичных компонентов.

## Быстрый старт
1. **Требования:** Node.js 18+, PNPM 9+.
2. **Установка зависимостей:**  
   ```bash
   pnpm install
   ``` Если удобнее npm, можно запустить те же команды, но lock-файл всё равно придётся перегенерировать (npm install, npm run dev).
3. **Запуск в dev-режиме:**  
   ```bash
   pnpm dev
   ```
   Проект поднимется на `http://localhost:3000`. Готовые аккаунты: `teacher@school.com` / `password123`, `student@school.com` / `password123`.
4. **Тесты:**  
   ```bash
   pnpm test
   ```
5. **Продакшн-сборка:**  
   ```bash
   pnpm build && pnpm start
   ```

## Деплой
Проект развёрнут на Vercel: https://school-lms-demo.vercel.app.

## Структура проекта
- `app/` — страницы Next.js (главная лента, login/register/profile/portfolio) + глобальный лейаут с подключением `ToastProvider`.
- `components/` — UI-модули: карточки постов, фильтры, меню профиля/мобильное меню, тосты, модальные окна для CRUD операций.
- `components/ui/` — переиспользуемые атомарные элементы (base toast, inputs, кнопки).
- `hooks/` — утилитарные React-хуки (`use-toast`, `use-mobile`) для уведомлений и адаптива.
- `lib/` — доменная логика: `mock-data`, `types`, `store` с фильтрами и CRUD над постами, `auth-store` для аутентификации, `utils`.
- `public/` — статика (иконки, изображения).
- `styles/` — глобальные стили Tailwind/PostCSS.
- `tests/` — юнит- и компонентные тесты Vitest + Testing Library, `setup.ts` подключает `@testing-library/jest-dom`.

## Что улучшено
- Добавлен `ToastProvider`/`ToastContainer`, чтобы пользователь мгновенно видел результат действий (создание/редактирование поста, ошибки входа).
- Внедрена фильтрация в `lib/store.ts` с возможностью показывать только свои посты, что ускоряет поиск релевантных записей.
- Создан `useIsMobile` для синхронизации десктопной и мобильной навигации без дублирования логики.
- Покрыты тестами ключевые компоненты (`post-card`, `create-post-modal`, `login-page`) для уверенности в рефакторингах.

## Что не успели / планы
- Подключить реальный backend/API вместо мок-данных — сейчас всё хранится в памяти `zustand`.
- Добавить загрузку файлов и хранение изображений достижений.
- Реализовать ролевые ограничения (кто может редактировать/удалять посты) и тонкую систему прав.
- Расширить тесты end-to-end и визуальные регресс-тесты.

## Принятые решения
- **Next.js App Router + Vercel**: быстрый SSR и простая публикация.
- **Zustand + persist**: минималистичное состояние без Redux, хранение auth-сессии в `localStorage`.
- **Radix UI + собственные компоненты**: единый UX-паттерн для модалок, тултипов и тостов.
- **Vitest + Testing Library**: быстрые компонентные тесты, схожие с React Testing Library API.
- **Полная локализация интерфейса**: весь текст на русском для целевой аудитории (учителя/ученики).
